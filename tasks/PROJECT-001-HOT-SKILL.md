## 🔄 双轨架构说明

```
【开发轨道】另一个 Agent（对话 A）
  └── 负责：技术开发、代码实现、调试迭代
  └── 产出写入：team/deliverables/hot-skill/

【管理轨道】我们团队（对话 B / 此处）
  └── 负责：评估质量、管理进度、协调资源
  └── 读取：team/deliverables/hot-skill/ 下的产出
  └── 评估输出：team/reports/ 下的各类评估报告
```

**两条轨道并行，通过共享目录 team/deliverables/hot-skill/ 同步。**

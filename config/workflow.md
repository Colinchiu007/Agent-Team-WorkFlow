# ⚙️ 工作流程规范

## 一、任务生命周期

```
创建 → 分配 → 执行 → 汇报 → 审核 → 关闭
```

### 1. 创建（Task Creation）
- **来源**：你的指令 → CEO 拆解
- **格式**：写入 `team/tasks/TASK.md`
- **包含**：任务名称、负责人、截止时间、优先级、关联 Agent

### 2. 分配（Assignment）
- **方式**：CEO 通过 `sessions_send` 向对应 Agent 发指令
- **通知**：同步更新 `team/tasks/TASK.md` 中任务状态
- **沟通**：Agent 接收指令，明确交付标准

### 3. 执行（Execution）
- Agent 在隔离 session 中执行任务
- 执行过程中记录关键进展到 `team/memory/TEAM.md`
- 遇到阻塞立即通过报告反馈

### 4. 汇报（Reporting）
- Agent 完成后，将报告写入 `team/reports/[role]-report.md`
- 报告格式见下方《报告规范》

### 5. 审核（Review）
- CEO 读取各 Agent 报告
- 审核质量、完整性、风险
- 有问题打回重做，没问题继续

### 6. 关闭（Close）
- 更新 `team/tasks/TASK.md` 任务状态为 Done
- 结果汇总写入 `team/reports/summary.md`
- 向你呈现最终结果

## 二、报告规范

### 每份报告必须包含

```markdown
# [角色] 汇报 - YYYY-MM-DD

## 📋 任务执行情况
- 任务名称：
- 执行时间：
- 执行状态：✅ 完成 / ⚠️ 部分完成 / ❌ 未完成

## 📝 执行详情
（详细描述做了什么、如何做的、输出物）

## 📊 交付物
（文件路径 / 产出清单）

## ⚠️ 风险与问题
（如有）

## 🔜 下一步行动
（如果有后续任务）
```

## 三、Cron 监督机制

### 每日站会（工作日 09:00）
```json
{
  "name": "team-daily-standup",
  "schedule": { "kind": "cron", "expr": "0 9 * * 1-5", "tz": "Asia/Shanghai" },
  "payload": { "kind": "systemEvent", "text": "【站会】各 Agent 提交每日进度报告到 team/reports/" }
}
```

### 周报汇总（每周一 09:30）
```json
{
  "name": "team-weekly-report",
  "schedule": { "kind": "cron", "expr": "0 9 * * 1", "tz": "Asia/Shanghai" },
  "payload": { "kind": "systemEvent", "text": "【周报】CEO 汇总本周各 Agent 工作，生成 team/reports/summary.md" }
}
```

## 四、异常处理流程

```
发现问题
    ↓
判断严重程度
    ├── 低：Agent 自主处理，记录到报告
    ├── 中：CEO 协调解决，更新任务
    └── 高：上报你确认，同时通知受影响方
```

### 异常分级标准

| 级别 | 定义 | 处理方式 |
|------|------|----------|
| 🔵 低 | 不影响整体进度 | Agent 自主处理，报告备注 |
| 🟡 中 | 可能影响里程碑 | CEO 介入，调整计划 |
| 🔴 高 | 影响整体目标达成 | 上报你决策 |

---

## 五、沟通约定

1. **指令入口**：只有一个——你向 CEO 下达指令
2. **输出出口**：只有一个——CEO 向你汇报结果
3. **Agent 间通信**：通过共享文件（`team/reports/`、`team/memory/`）非实时协作
4. **实时协作**：通过 `sessions_send` 向指定 Agent 发送即时消息

---

## 六、任务优先级

| 优先级 | 定义 | 响应时间 | 处理方式 |
|--------|------|----------|----------|
| P0 | 紧急重要 | 即时处理 | 暂停其他所有任务全力处理 |
| P1 | 重要不紧急 | 当天内 | 按计划执行，高优先级 |
| P2 | 一般任务 | 3 天内 | 正常排队 |
| P3 | 探索/优化 | 有余力时 | 低优先级，可延后 |

---

## 七、交付标准

所有交付物需满足：

- ✅ **完整**：包含所有约定的内容
- ✅ **准确**：信息无误，符合需求
- ✅ **可执行**：如果是方案，需可直接落地执行
- ✅ **有记录**：写入对应报告，纳入团队知识库

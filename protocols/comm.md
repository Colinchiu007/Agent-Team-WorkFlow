# 🤝 协作通信协议

## 一、通信原则

### 即时通信 vs 异步通信

| 类型 | 工具 | 适用场景 |
|------|------|----------|
| 即时 | `sessions_send` | 紧急指令、实时讨论、需要澄清的问题 |
| 异步 | 共享文件（team/reports/） | 任务交付、进度汇报、非紧急通知 |

### 通信优先级

```
P0 紧急 → sessions_send（立即）
P1 重要 → sessions_send（当天内）
P2 一般 → 写入报告文件（任务完成后）
```

## 二、Agent 间通信规范

### CEO → Agent 指令格式

```
【指令】
任务：xxx
负责人：[角色]
截止时间：YYYY-MM-DD HH:mm
优先级：P0/P1/P2/P3
完成标准：
1. xxx
2. xxx
3. xxx
交付物：请将报告写入 team/reports/[role]-report.md
```

### Agent → CEO 汇报格式

```
【汇报】
角色：[角色名]
任务：[任务名]
执行状态：✅ 完成 / ⚠️ 部分完成 / ❌ 未完成
用时：xxx

执行详情：
[详细描述做了什么、如何做的]

交付物：
[文件路径或产出清单]

风险与问题：（如有）
[描述]

下一步行动：（如有）
[描述]
```

### Agent → Agent 通信

**直接通信**：通过 `sessions_send(sessionKey, message)` 直接发消息

**共享文件通信**：
- 写入位置：`team/reports/`
- 读取位置：`team/tasks/`, `team/memory/`

## 三、文件命名规范

### 报告文件
```
[role]-[类型]-YYYY-MM-DD.md
示例：
cto-progress-2026-05-11.md
pm-wbs-小程序的开发-2026-05-11.md
coo-weekly-2026-W20.md
```

### 任务文件
```
TASK-[序号]-[任务名].md
示例：
TASK-001-小程序开发.md
TASK-002-618运营活动.md
```

### 交付物
```
[项目名]/[模块]/[文件名]
示例：
apps/reading-notes/frontend/index.html
docs/小程序技术方案.md
```

## 四、状态标识规范

### 任务状态
| 标识 | 含义 |
|------|------|
| 🔵 待领取 | 任务已创建，等待分配 |
| 🔄 进行中 | 正在执行 |
| ⚠️ 遇阻 | 执行中遇到问题，需要支持 |
| ✅ 完成 | 任务完成，待审核 |
| ✅ 已审核 | CEO 审核通过 |
| ❌ 取消 | 任务取消 |

### 优先级
| 标识 | 含义 |
|------|------|
| 🔴 P0 | 最高，暂停一切立即处理 |
| 🟠 P1 | 高优先级，当天完成 |
| 🟡 P2 | 一般，3天内完成 |
| 🔵 P3 | 低优先级，有余力处理 |

### 风险等级
| 标识 | 含义 |
|------|------|
| 🔴 高风险 | 影响核心目标达成 |
| 🟡 中风险 | 影响里程碑 |
| 🔵 低风险 | 不影响整体进度 |

## 五、会议/站会规范

### 每日站会（工作日）
- **时间**：09:00（通过 Cron 自动触发）
- **参与**：PM 主持，各 Agent 报告
- **输出**：`team/reports/standup-YYYY-MM-DD.md`
- **时长**：5 分钟（书面）

### 周报会议（每周一）
- **时间**：09:30
- **参与**：CEO 汇总，生成周报
- **输出**：`team/reports/summary-YYYY-WXX.md`
- **内容**：各 Agent 周报 + 本周计划

### 紧急会议
- 由 CEO 发起，通过 `sessions_send` 即时通知
- 针对 P0 级别问题

## 六、知识共享规范

所有 Agent 的工作成果，应沉淀到以下位置：

| 类型 | 存放位置 |
|------|----------|
| 任务说明 | `team/tasks/` |
| 阶段性报告 | `team/reports/` |
| 项目文档 | `team/deliverables/[项目]/` |
| 团队记忆 | `team/memory/TEAM.md` |
| 通用知识 | `team/memory/` |

## 七、冲突解决协议

```
步骤 1：发现冲突（角色 A vs 角色 B）
↓ 角色 A 或 B 报告给 CEO
↓ 步骤 2：CEO 评估冲突等级
   ├── 低影响 → CEO 裁定，执行
   └── 高影响 → CEO 上报你决策
↓ 步骤 3：执行决策，更新任务状态
↓ 步骤 4：记录到 team/memory/TEAM.md
```
